## Question

What is the simplest coherent user journey from Overview or Project through Collection Request, generated Workflow, Test/Live Canvas, published Data Feed, Query Request, and actionable Inbox without exposing infrastructure-first configuration or duplicating global modules inside a Project?

## Resolution

Resolved through live review in [Collection-to-consumption journey prototype resolution](assets/prototype-collection-to-consumption-journey.md). Use a familiar Dify-style create, compose, test, publish, monitor, and iterate journey around the existing operations dashboard and a deeper packaged Node system. Collection is the first complete vertical, not the product boundary; published Workflows may become pages, chat, CLI, APIs, Tools, Automations, batch jobs, dashboard components, Artifacts, or Data Feeds.
